1. 市場環境認識（L01: ドミナンス・マトリクス）

1.1 前提: なぜ市場を6つに分類するのか

このBotは市場を6つの状態（パターン）に分類します。判定に使う3つの指標:

指標何を見ているか↑ の条件↓ の条件→ の条件
BTC価格 24h変動率市場全体の方向+1.0%以上-1.0%以下-1.0%〜+1.0%
BTC.D（ドミナンス）変化資金がBTCに集中か、アルトに流れているか+0.3%pt以上-0.3%pt以下-0.3%pt〜+0.3%pt
Total MC 24h変動率市場全体にお金が入っているか出ているか+1.0%以上-1.0%以下-1.0%〜+1.0%
閾値の出典: src/signals/regime.py L22-L24
THRESHOLD_BTC = 1.0 / THRESHOLD_TOTAL = 1.0 / THRESHOLD_DOM = 0.3

なぜこの3指標か:

  • BTCが上がっているか下がっているか = 市場全体の方向。BTCは仮想通貨市場の基軸であり、全体の温度計
  • BTC.Dが上がっているか下がっているか = 資金がBTCに集中しているか、アルトに流れているか
  • Total MCが上がっているか下がっているか = 市場全体にお金が入っているか出ているか（フィアットとの出入り）

2.1 ファンネル構造の意味

   MEXC先物全銘柄（約870銘柄）
          │
          ▼
   ┌────────────────────────────┐

  • Tier1のみ通過 = 「面白い動きだが、流動性リスクがある」
  • Tier1+Tier2通過 = 「面白い動きで、かつ安全にトレードできる」
  • どちらも通過しない = 「今は触る必要なし」

3. Bot ステータスの全種類と意味

3.1 ステータス一覧

ステータス英語キー意味次のアクション
待機中waiting発動条件を満たしていない。市場を監視し続けているなし。条件が揃うまで待つ

3.2 各Bot ステータス遷移条件

Bot-Alpha（極限一撃モード）

待機中 → 発火圏内: Fear < 15

発火圏内 → 稼働中: 3条件同時成立

  1. Fear < 10 (bot_alpha_engine.py L34)
  2. BTC 24h ≤ -1.0% (bot_alpha_engine.py L35)
  3. BTC.D変化 ≤ -0.5%pt (bot_alpha_engine.py L36)

稼働中 → シグナル: BTC相関 < 0.5 AND α ≥ 3.0% → Long, 3x, TP8%/SL3%

Bot-Surge（日常循環モード）

待機中 → 稼働中: 2条件同時成立

  1. Fear 25〜45 (bot_surge_engine.py L92)
  2. BTC 24h ≤ 0% (bot_surge_engine.py L93)

シグナル経路1: abs(btc_divergence) > 3.0% AND RSI > 50

シグナル経路2: セクターカスケード（リーダーイベント後 lag_days × 80-120%）

Bot-MeanRevert（過熱ショート）

待機中 → 稼働中:

  1. Fear 50〜80
  2. MA20上方乖離 > 15%
  3. 直近2日出来高 > 20日平均 × 1.2（売り圧力兆候）

→ Short, 2x, TP6%/SL3%

Bot-WeakShort（弱いアルトショート）

待機中 → 稼働中:

  1. Fear 50〜75
  2. BTC日次 ≥ +1.0%

ターゲット: BTCリターン - アルトリターン ≥ 3.0% → Short, 2x, TP5%/SL2.5%

Bot-Sniper（精密狙撃）

待機中 → 稼働中: 4条件同時

  1. Fear < 30
  2. BTC日次 ≤ -3.0%
  3. 出来高 ≥ 5.0x平均
  4. BTC相関 < 0.3

→ Long, 10x, TP15%/SL5%

Bot-Scalp（BB+RSI短期回転）

常時稼働（Fear制限なし）

Long: BB(20,2σ)下限タッチ + RSI(14) ≤ 30

Short: BB(20,2σ)上限タッチ + RSI(14) ≥ 70

→ 10x, TP8%/SL3%

4. トリガー発動後の動作フロー

4.1 トリガーとは

トリガー発動とは: 市場で「普段と違う大きな変化」が起きたことを検知する仕組みです。

重要な区別:

トリガー = 「今すぐレポートを見てください」というアラート

トリガー ≠ 「今すぐ買ってください」という指示

4.2 5つのトリガー条件

#トリガー閾値何が起きたかCD
1BTC 1h価格変動±5.0%フラッシュクラッシュ or 急騰2h
2BTC 4h価格変動±8.0%大規模な方向転換2h
3Fear&Greed変動±10pt市場心理の急変2h
4BTC.D日次変動±1.5%pt大規模な資金移動2h
5OI 1h変動±15.0%大量清算の発生2h
閾値出典: config/settings.yaml L305-310
クールダウン: 条件ごとに独立管理（cooldown_seconds: 7200）
異なるトリガーは同時に発火可能

4.3 発動後の動作

トリガー条件検出
    │
    ├── 妥当性チェック（異常値でないか確認）
    │   BTC価格: $1K-$500K 範囲外 → トリガー除外
    │   BTC.D: 30-80% 範囲外 → トリガー除外
    │   Fear: 0-100 範囲外 → トリガー除外
    │
    ├── クールダウンチェック（同トリガーが2h以内に発火済みか）
    │
    └── 緊急レポート送信
        ├── Telegram通知
        │   - トリガー理由
        │   - BTC価格, Fear, BTC.D, レジーム
        │   - Tier1/Tier2通過銘柄一覧（Top20）
        │   - Bot状態, ポジション, フィードバック統計
        └── 次サイクルで通常スキャン続行

5. 処理フロー図（詳細版）

→ score > 10 かつ L10(板厚) VETO なし → 約65-88銘柄
