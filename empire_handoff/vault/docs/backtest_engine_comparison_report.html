2. 2つのエンジンの処理フローの違い

BacktestEngine（旧）
1 シグナル検知（日付N）
Fear&Greed、MA20乖離等の条件を判定
2 翌日始値でエントリー（日付N+1）
look-ahead bias回避のため正しい
3 同日の高値/安値でTP/SL判定（日付N+1）
エントリー日にそのまま即判定 → ほぼ即日決済
→ 平均保有期間: 0.1日
翌日に持ち越さない → 含み損蓄積なし → 最大ドローダウン小
simulate_trades_fast（グリッドサーチ）
1 シグナル検知（日付N）
同じロジックで同じシグナルを収集
2 翌日始値でエントリー（日付N+1）
同じ価格でエントリー
3 翌日以降に初めてTP/SL判定（日付N+2〜）
エントリー日の判定は翌ループ → 1日以上保有確定
→ 平均保有期間: 1.1日
翌日に持ち越す → 含み損が蓄積 → 最大ドローダウン大

なぜリターンはグリッドサーチの方が高いのか
旧エンジンは即日TP判定のため、テイクプロフィット価格で決済される割合が高い。一方グリッドサーチは翌日判定のため、テイクプロフィットを超えた価格で決済されることがある（翌日の始値がTPを超えている場合）。また、ストップロスに引っかからず翌日以降にプラスに転じるケースも含まれる。
リターンが高い一方で、保有中の含み損がエクイティカーブに反映されるため最大ドローダウンが悪化する。

5. どちらの数値が「正しい」のか

旧エンジンが正しいケース

日足のTP/SL判定で日中の高値/安値がヒットした場合、指値注文を事前に出しておけば約定する可能性がある。

ただし以下の条件が必要:

  • エントリー直後にTP/SL指値を設定
  • 十分な流動性（板が薄いとスリッページ大）
  • 取引所のAPI遅延が最小
  • 高値/安値のヒゲ部分で実際に約定できる
グリッドサーチが正しいケース

成行注文でエントリーし、TP/SLの判定が次回スキャンまで行われない場合。

以下の状況でこちらに近づく:

  • スキャン間隔が長い（数時間〜1日）
  • 指値でなく成行でTP/SL判定
  • API遅延やエラーでTP/SL発注が遅れる
  • 日中の瞬間的な高値/安値では約定できない
現実的な推定値
現実のドローダウンは両者の中間にあると考えるのが妥当です。
