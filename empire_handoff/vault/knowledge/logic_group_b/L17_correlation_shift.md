# L17: CACS（Correlation Shift / 相関シフト）

## 概要
BTCが下落しているのに耐えている（相関が外れている）銘柄をAlphaとして検出する。
相関係数 < 0.5 の銘柄をBTC脱却（独自走）候補として識別する。
L02（聖域判定）と組み合わせた場合に最強のシグナルとなる。

## 定義・目的
ほとんどのアルトコインはBTCと高い正の相関を持つ。しかし、独自の材料（実需・資金流入）を持つ銘柄はBTC下落時にも価格を維持または上昇する。この「相関の崩れ」を検出することで、真のAlpha銘柄を発見する。

## 判定ロジック
- **相関係数計算**: 各銘柄 vs BTC の30日ローリング相関係数
- **Alpha検出**: 相関係数 < 0.5 → BTC脱却（独自走）の可能性
- **最強条件**: BTC下落 + 該当銘柄横ばい/上昇 → 真のAlpha
- **偽陽性注意**: 単に流動性が低くて動かない銘柄は除外（L03/L10で併用フィルター）

## データ要件
- **各銘柄の日次リターン**: ccxt_ohlcv
- **BTCの日次リターン**: ccxt_ohlcv
- **過去30日分のデータ**
- **更新頻度**: 日次
- **MEXC互換性**: 可

## データプロキシ
該当なし（直接計算可能）

## 実装ヒント
- 全銘柄の日次リターンとBTCリターンの30日ローリング相関を計算
- pandas `rolling(30).corr()` で効率的に計算
- 相関 < 0.5 の銘柄を抽出
- L02（聖域判定）通過 + L17（低相関） = Alpha候補
- Tier: 1
- 計算コスト: 中量（870銘柄 x 30日相関計算。ただし日次でキャッシュ可能）

## 他ロジックとの関係
- **依存するロジック**: なし（独立計算）
- **連携するロジック**: L02（聖域付近 + 低相関 = 最強シグナル）、L03（出来高伴走確認）
- **矛盾・注意点**: 相関は時間軸依存。30日相関が低くても、直近1週間で相関復帰している可能性あり

## トレード実行への適用
- **スクリーニング**: 低相関銘柄をAlpha候補として優先スクリーニング
- **エントリー**: BTC下落 + 該当銘柄横ばい + 聖域付近 → 強力な買いシグナル
- **エグジット**: 相関が0.8以上に復帰 → 独自性消失、BTC連動リスク増大
- **スイング（数日〜数週間）**: 低相関を維持している間はスイング保有継続
- **短期利確（10%+）**: BTC反発時に相関復帰する前に短期利確

## 注意事項
- ⚠️ 相関 < 0.5 の閾値はalpha_logicとsurf_cryptoの記載に基づくが、バックテストによる最適化が必要
- ⚠️ 流動性が低い銘柄は自然と相関が低くなるため、L03/L10との組み合わせフィルターが必須
