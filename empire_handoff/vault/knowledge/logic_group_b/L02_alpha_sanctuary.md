# L02: Alpha Sanctuary（髭下判定 / 聖域サポート）

## 概要
**最重要ロジック**。定義された「10/10安値（聖域）」を現在価格が割っていないかを判定する。
割っていれば即時除外（Critical Fail）。聖域付近での反発は高スコア。
全21ロジックの中で最も強力なフィルターであり、Tier 1の最初に実行される。

## 定義・目的
2025年10月10日の髭下（wick low）を「聖域サポート」と定義。この価格帯はマーケットが一度テストし跳ね返った実績のあるレベルであり、ここを割ることは構造的な崩壊を意味する。聖域付近での反発は高スコア（買いシグナル）。BIO +450%の例のように、聖域からの回帰ポテンシャルが大きい。

## 判定ロジック
- **即時除外（Critical Fail）**: 現在価格 < 10/10安値 → スクリーニングから完全除外
- **高スコア**: 現在価格が10/10安値の+5%以内で反発 → 聖域サポートの強さを確認
- **乖離率計算**: `dist_from_1010_pct = (現在価格 - 10/10安値) / 10/10安値 * 100`
- **Alpha判定**: BTC下落時（Fear & Greed低位）に対BTC騰落 > 0 を維持 = 真の強者

## データ要件
- **現在価格**: ccxt_ticker（MEXC Futures）
- **10/10安値**: 手動設定 or 過去データから計算（asset_master.csv の `low_1010_price`）
- **対BTC騰落率**: ccxt_ohlcv（各銘柄/BTC比率の変化率）
- **更新頻度**: 毎分（価格）/ 初回設定時のみ（10/10安値）
- **MEXC互換性**: 価格は可。10/10安値は過去データから算出

## データプロキシ
- 10/10安値が取得困難な新規上場銘柄: 上場来安値を代替聖域とする
- 対BTC騰落率: `(銘柄変化率 - BTC変化率)` で簡易計算

## 実装ヒント
- asset_master.csvに `low_1010_price` カラムを事前設定
- Tier 1の最初に実行（最も軽量かつ最大の足切り効果）
- 聖域割れ銘柄は以降のTierに一切進めない（早期リターン）
- 乖離率はAnalysisPayloadに含めてTier 2以降に引き渡し
- Tier: 1
- 計算コスト: 軽量（単純な価格比較）

## 他ロジックとの関係
- **依存するロジック**: なし（独立判定）
- **連携するロジック**: L17（CACS）と組み合わせて「聖域付近 + BTC非相関」の銘柄を最優先
- **矛盾・注意点**: 10/10安値は固定値だが、市場環境変化で聖域自体を更新する判断が必要になる可能性あり
